# 🧠 Examen Práctico de Lógica en Java – Liquidación de Proyectos de Programador

## 📋 Descripción General
Este proyecto fue desarrollado como parte del **examen del módulo de Lógica de Programación**, en el que se partió de una versión con errores y se trabajó en su **análisis, depuración, corrección y la comprobación de la funcionalidad completa del codigo**.

---

## 🎯 Objetivo del Proyecto
- Analizar la totalidad del codigo.
- Identificar los errores que no permitian la ejecucion del codigo.
- Depurar el codigo, corrigiendo errores de sintaxis, eliminando y anexando las lineas necesarias.
- Compilar el codigo, para comprobar su total funcionalidad.
- Hacer un buen versionamiento del codigo y un pull request.

---

## ✅ Funcionalidades Completas
- Captura de datos personales (nombre, edad, ciudad, país, etc.).
- Cálculo automático de la tarifa por hora según:
  - Nivel de experiencia (Junior/Senior).
  - Años de experiencia.
- Registro de tres clientes y sus proyectos.
- Cálculo del pago por cliente (horas trabajadas + bono).
- Cálculo del subtotal, descuento (3%) e impuesto (9%).
- Captura automática de la fecha de liquidación con `LocalDate`.
- Generación de un **reporte final completo y bien estructurado**.

---

## 🐞 Principales Errores Encontrados y Solucionados

| Error o Problema | Descripción | Solución Aplicada |
|------------------|-------------|--------------------|
| Error principal no permitia la compilacion del codigo |Una mal estructura desde el inico en el MAIN no permitia ejecucion| Se aplico un cambio en la estrctura y una pequeña correcion en la sintaxis para permitir la compilacion |
| Se utilizo incorrectamente el localDate | no se hacia la captura de la informacion de la fecha | Se impotaron los componentes para su correcto funcionamiento |
| Errores graves en las Sintaxis | visiblemente se evidensiava la ausenscia de componetes como: ( ; - {} - etc ) | Se agregan los componentes faltantes a la sintaxis del codigo para su correcta compilacion  |


---

## 🔄 Mejoras Agregadas
- Uso de la clase `LocalDate` para capturar la fecha actual de liquidación.
- Bonificación individual por cada cliente.
- Cálculos exactos y expresados en consola con formato entendible.
- División clara del reporte final con encabezados.
